# Comparison: MySQL on VM vs. Managed MySQL on Cloud SQL
I would use Managed MySQL on Cloud SQL over VM. The Managed MySQL was much more easier to create an instance and add firewall ports. There was less steps than the VM. However it did take a longer time to load the creation of the instance. After creating the instances, VM took longer as there was more steps to the process where you had to go into the SSH multiple times to update the OS system, instal mysql and nano. You also had to fo ingto the nano to change the bind address manually to 0.0.0.0/0. Then it required a restart of the system before u could log in and add in the data. Howwever for me, it was easier to set up afterwards. While doing the Managed MySQL, I ran into problems running the python code. I think the password I created was confusing the terminal. Since google shell had requirements to what the password had to include. Although that is good for safety reasons, it was very annoying for me to deal with. For a small student app, I would choose Managed Cloud SQL as it is more beginner friendly and helps students create the instance and database easily without much inputs needed. It is also more low cost as mine was $0.01 compared to $13.
